# legacy 特殊技能梳理

## 目标

梳理 `jyx-legacy-dll` / `jyx-legacy-data` 中“特殊技能”体系的定义来源、运行时入口、逐技能效果、获得路径与迁移观察，作为后续新版内容建模与战斗迁移的参考。

本文只基于当前工作区内可见文件整理，不依赖工作区外资源。

## 范围与来源

本次梳理主要基于以下文件：

- `jyx-legacy-data/special_skills.xml`
- `jyx-legacy-data/roles.xml`
- `jyx-legacy-data/items.xml`
- `jyx-legacy-data/storys.xml`
- `jyx-legacy-dll/JyGame/SpecialSkill.cs`
- `jyx-legacy-dll/JyGame/SpecialSkillInstance.cs`
- `jyx-legacy-dll/JyGame/SkillBox.cs`
- `jyx-legacy-dll/JyGame/SkillCoverType.cs`
- `jyx-legacy-dll/JyGame/AttackLogic.cs`

说明：

- `special_skills.xml` 中一共定义了 27 个特殊技能。
- `SpecialSkill` 是静态定义，`SpecialSkillInstance` 是角色持有的运行时实例。
- 战斗时特殊技能主要经 `AttackLogic.Attack(...)` 分发。
- `AttackLogic.cs` 末尾存在 `AttackLogic_extendSpecialSkill` Lua 扩展入口，但当前工作区内没有对应 legacy Lua 源文件，因此本文只能把它视为“潜在扩展点”，不能把其行为当作已证实实现。

## 体系结构

### 定义层

`special_skills.xml` 为每个特殊技能提供以下核心字段：

- `name`
- `info`
- `costMp`
- `costball`
- `cd`
- `castsize`
- `coversize`
- `covertype`
- `hitself`
- `animation`
- `audio`
- `buff`

### 运行时层

`SpecialSkillInstance` 只是一层薄包装，负责把运行时查询转发到静态定义：

- 名字、图标、冷却、消耗、施法范围、覆盖范围、动画、音效都来自 `SpecialSkill`
- `SkillType` 固定为 `Special`
- 描述文本由 `info + 范围/消耗 + buff` 拼接

### 战斗层

特殊技能的真实效果来源有三种：

1. `AttackLogic.cs` 显式硬编码。
2. `special_skills.xml` 的 `buff` 字段通过通用 buff 结算生效。
3. `AttackLogic_extendSpecialSkill` Lua 扩展补充。

因此不能只看 `info` 文案判断真实效果，必须以 `AttackLogic.cs` 和可见 buff 为准。

## 覆盖类型

`SkillCoverType` 在 legacy 中的含义如下：

- `0`: 点攻击 `NORMAL`
- `1`: 十字攻击 `CROSS`
- `2`: 米字攻击 `STAR`
- `3`: 直线攻击 `LINE`
- `4`: 面攻击 `FACE`
- `5`: 扇形攻击 `FAN`
- `6`: 环状攻击 `RING`
- `7`: 对角线攻击 `X`
- `8`: 身前攻击 `FRONT`

## 实现观察

### 1. 特殊技能不是纯数据驱动体系

虽然 `special_skills.xml` 提供了统一字段，但真实效果并不统一靠数据解释：

- 一部分技能只是“释放 buff”，适合数据化。
- 一部分技能带有明确的战场位移和资源操作，依赖硬编码。
- 一部分技能同时混合了硬编码与 buff。

因此如果要迁移到现有新版内核，不适合按“把 XML 原样搬过来”处理，而应先做语义拆分。

### 2. 文案和代码并不总是一致

几个明显例子：

- `解毒` 文案是“多强的毒都清除”，但可见代码只删除 `中毒`。
- `笑傲江湖曲` 文案写“增强全体攻击力”，代码里也确实只施加攻击强化。
- `武穆兵法` 文案写“增强所有队友能力”，代码实际给的是攻击、防御、神速攻击三种 buff。

后续若重建内容，不应把 `info` 当作规范真相，而应重新定义正式语义。

### 3. 若迁移到当前新版，建议按机制而不是按 legacy 名称复刻

比较合理的拆法是：

- 单体治疗类：华佗再世
- 单体真伤爆发类：一刀两断、火枪、襄儿的心愿、同归剑法、天地同寿、腐尸毒
- 自身状态类：打鸡血、诗酒飘零、凌波微步、雪遁步行、梯云纵、溜须拍马、易容术
- 群体净化/驱散类：清心普善咒、阿碧的歌声、吴侬软语
- 团队强化类：笑傲江湖曲、武穆兵法
- 控制与位移类：撒石灰、过肩摔、神龙摆尾、六合劲
- 全场异常类：飞星术

也就是说，应该先抽象出一组稳定的 battle effect 原语，再把 legacy 名称投影上去。

### 4. `吴侬软语` 存在技能与天赋同名语义冲突

当前可见实现里：

- 特殊技能 `吴侬软语` 本身负责敌方群体驱散。
- 若施法者同时拥有天赋 `吴侬软语`，则该技能额外附带群体晕眩。

这说明 legacy 实际是在做“技能 + 天赋联动”，而不是一个单独技能的固定效果。新版若照搬名称，建议把它拆成：

- 基础技能效果
- 天赋修正效果

### 5. `腐尸毒` 与 `飞星术` 的内容闭合度不足

这两个技能的战斗逻辑明确存在，但当前可见获得路径不闭合。它们是后续迁移时必须先确认的两个问题点：

1. 是否保留为正式可获得技能。
2. 若保留，应由谁获得、通过什么内容路径获得。
3. 是否需要改成内功/门派专属派生效果，而不是独立特殊技能。

## 结论

legacy 的特殊技能体系规模不大，只有 27 个，但混合了：

- 纯 buff 技
- 资源控制技
- 位移技
- 群体净化/驱散技
- 团队强化技
- 高代价爆发技

从第一性原则看，它更像“角色专属战斗动作集合”，而不是可以直接并入新版统一技能模型的一组同构定义。

后续如果要在当前项目中重建这部分内容，建议先做两件事：

1. 把这 27 个技能映射到新版可支持的 effect 原语集合。
2. 先决定哪些技能需要保留为正式可学内容，哪些只保留为剧情/NPC 专属动作。
