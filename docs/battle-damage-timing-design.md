# 战斗伤害 Timing 设计

## 目标

本文只定义伤害相关 timing 的职责边界，不讨论完整战斗状态机。

本文目标：

- 明确每个 timing 允许放哪些类型的效果
- 明确普通伤害与真实伤害分别走哪些 timing
- 避免把“命中前拦截”“防御修正”“扣血后触发”混在同一个 timing

本文是后续重构目标，不要求兼容当前运行时代码。

## 核心原则

- 命中判断与伤害数值结算分开
- 防御体系与受伤事实分开
- 真实伤害绕过防御，不绕过受伤事实
- timing 按职责划分，不按某个具体技能命名

## 四个核心 Timing

### BeforeHitResolved

语义：

- 这一下是否算命中
- 这一下是否被直接拦截为 MISS / 无效命中

允许放的效果类型：

- 闪避
- 必中
- 破闪避
- 易容
- 溜须拍马
- 其他“命中是否成立”的判定

不应放的效果类型：

- 按攻击/防御/暴击修改伤害值
- 依赖“本次伤害数值”的减伤
- 受伤后触发

典型结果：

- 命中
- MISS
- 命中成立但禁止后续附加效果

当前适合挂载的 effect 原语：

- `cancel_hit`
  - 用于闪避、易容、溜须拍马这类直接让本次命中失效的效果
- `remove_buff`
  - 仅在“命中判定成立后立刻破功”这类场景使用
  - 典型例子是 `易容` 奇袭后移除自身 buff
- 其他 effect 原语默认不建议挂在这一层

### BeforeDamageCalculation

语义：

- 命中已经成立
- 现在进入普通伤害公式的数值修正

允许放的效果类型：

- 攻击乘区
- 防御乘区
- 暴击率修正
- 暴击倍率修正
- `final_damage` 修正
- 常规增伤
- 常规减伤
- 乾坤大挪移
- 宝甲
- 神经病
- 鲁莽

不应放的效果类型：

- 闪避
- 斗转星移
- 不老长春
- 受伤回怒

说明：

- 该 timing 只服务普通伤害
- 真实伤害不走这一层
- `final_damage` 只表示普通伤害公式内部的最后一层纯数值修正
- `final_damage` 不负责分摊、转移、溢出转资源、清零并回血这类带副作用的落血改写

当前适合挂载的 effect 原语：

- `modify_damage_context`
  - 这是这一层的主力原语
  - 常见字段：
    - `source_attack`
    - `source_attack_low`
    - `source_attack_high`
    - `target_defence`
    - `critical_chance`
    - `critical_multiplier`
    - `final_damage`
- 不建议在这一层挂：
  - `cancel_hit`
  - `add_rage`
  - `set_rage`
  - `set_action_gauge`
  - `add_hp`
  - `add_mp`

### BeforeDamageApplied

语义：

- 命中已经成立
- 当前已经得到一个伤害值
- 但目标还没有真正扣血

允许放的效果类型：

- 依赖“本次伤害值”的防御性改写
- 伤害封顶
- 伤害减半
- 伤害分摊
- 伤害转移
- 斗转星移
- 不老长春
- 其他“拿到伤害值后，再决定怎么落血”的效果

不应放的效果类型：

- 闪避
- 攻击/防御/暴击的公式修正
- 受伤后触发

说明：

- 这一层本质上是“伤害落地前的防御与反制”
- 如果某个机制需要读取本次最终伤害值，但又会改写伤害落点，应放在这里

当前适合挂载的 effect 原语：

- 当前仓库里还没有专门为这一层收敛出完整 effect 集
- 现阶段可以预期后续主要会落到这里的 effect 方向：
  - 伤害封顶
  - 溢出转资源
  - 分摊 / 转移
  - 反弹 / 反制
  - 清零并回血
- 已有原语中，原则上只有“确实依赖最终伤害值并带副作用”的将来专用 effect 才应进入这一层
- `modify_damage_context` 不应继续承担这类职责

### OnDamageTaken

语义：

- 目标已经实际扣血
- 现在处理“受伤之后”的反应

允许放的效果类型：

- 受击回怒
- 受到伤害后触发被动
- 受伤叠层
- 受伤后回血/回蓝/回能量
- 受伤统计
- 受伤后的表现事件

不应放的效果类型：

- 闪避
- 命中拦截
- 普通伤害公式修正
- 斗转星移
- 不老长春

说明：

- 这一层看的是“最终实际损失了多少 HP”
- 如果伤害被完全拦截或被改写成 0，则不应进入这一层

当前适合挂载的 effect 原语：

- `add_rage`
  - 受击回怒
- `apply_buff`
  - 受伤后叠层或追加状态
- `add_hp`
  - 受伤后回血
- `add_mp`
  - 受伤后回蓝
- `remove_buff`
  - 受伤后消耗某个状态
- `remove_negative_buffs`
  - 极少数“挨打后清异常”机制
- `remove_positive_buffs`
  - 极少数“挨打后破状态”机制
- 原则上不建议在这一层挂：
  - `cancel_hit`
  - `modify_damage_context`

## 普通伤害与真实伤害

### 普通伤害

普通伤害的推荐管线：

1. `BeforeHitResolved`
2. `BeforeDamageCalculation`
3. `BeforeDamageApplied`
4. 扣血
5. `OnDamageTaken`

适用对象：

- 外功
- 招式
- 大部分依赖攻击/防御/暴击公式的伤害

### 真实伤害

真实伤害的推荐管线：

1. `BeforeHitResolved`
2. 跳过 `BeforeDamageCalculation`
3. 跳过 `BeforeDamageApplied`
4. 直接扣血
5. `OnDamageTaken`

适用对象：

- 火枪
- 襄儿的心愿
- 同归剑法
- 其他明确声明为真实伤害的技能效果

设计含义：

- 真实伤害绕过防御体系
- 真实伤害仍然算“真正受伤”
- 因此真实伤害会触发扣血后的受伤逻辑

## 效果归类规则

判断某个效果该挂在哪个 timing，可按下面的顺序判断：

1. 它是在判定“打中没打中”吗？
   如果是，放 `BeforeHitResolved`

2. 它是在改普通伤害公式里的攻击、防御、暴击或最终倍率吗？
   如果是，放 `BeforeDamageCalculation`

3. 它需要先拿到本次伤害值，再决定是否减伤、分摊、转移、反制吗？
   如果是，放 `BeforeDamageApplied`

4. 它要求目标已经真实掉血后才生效吗？
   如果是，放 `OnDamageTaken`

## 典型映射

### 命中前拦截

- 易容：`BeforeHitResolved`
- 溜须拍马：`BeforeHitResolved`
- 闪避类天赋：`BeforeHitResolved`
- 必中类天赋：`BeforeHitResolved`

### 普通伤害公式修正

- 乾坤大挪移：`BeforeDamageCalculation`
- 宝甲：`BeforeDamageCalculation`
- 神经病：`BeforeDamageCalculation`
- 鲁莽：`BeforeDamageCalculation`

### 落血前防御/反制

- 不老长春：`BeforeDamageApplied`
- 斗转星移：`BeforeDamageApplied`
- 分摊/替挡：`BeforeDamageApplied`
- 伤害封顶：`BeforeDamageApplied`
- 溢出转资源：`BeforeDamageApplied`

### 扣血后反应

- 受击回怒：`OnDamageTaken`
- 受到伤害后触发的被动：`OnDamageTaken`

### BeforeDamageApplied

应归入“已经得到伤害值，但尚未扣血”的 legacy 实现：

- 依赖最终伤害值，并伴随额外副作用的改写
  - `精明`（守方 500 封顶并把溢出转成 MP）
- 依赖最终伤害值的反制 / 转移 / 分摊
  - `斗转星移`
  - `真武七截阵`（守方改目标并把本次伤害压到 30%）
  - `五行阵`
  - `八卦阵`
- 依赖最终伤害值的“改写本次伤害”
  - `不老长春`
  - `黑天死炎`

备注：

- legacy 中这层与 `BeforeHitResolved` 部分效果写在同一大段尾流程里，但语义不同，迁移时应拆开

### OnDamageTaken

应归入“HP 已经真实扣除后”的 legacy 实现：

- 吸血 / 吸内 / 吸蓝
  - `血海魔功`
  - `血魔刀法`
  - `嗜血狂魔`
  - 角色 trigger `xi`
  - `北冥神功`
  - `鲲跃北溟`
  - `化功大法`
  - `吸星大法`
- 扣血后追加资源 / 节奏变化
  - `玄门罡气`（扣血后减少集气）
  - `暴躁`（受击回怒）
  - `斗魂`（受击回怒翻倍）
- 扣血后再增强本次 debuff / buff 结算
  - `墨家传人`
  - `毒系精通`
  - `毒圣`
  - `我是疯子`

备注：

- 这些效果都依赖“本次命中已经真正造成了后果”
- 它们不适合放到命中前拦截层，也不适合继续混在防御改写层里

### 关于 `final_damage`

`final_damage` 字段保留，但只用于普通伤害公式内部的末端纯数值修正。

适合放到 `final_damage` 的效果：

- `乾坤大挪移奥义`
- `乾坤大挪移`
- `宝甲`
- `神经病`
- `鲁莽`
- `易容` 奇袭增伤

不适合放到 `final_damage` 的效果：

- `精明` 这种“封顶并把溢出转成 MP”
- `斗转星移` 这种“反制并转嫁”
- `五行阵` / `八卦阵` 这种“分摊 / 替挡”
- `不老长春` 这种“把本次伤害清零并回血”

判断标准：

- 如果只是改一个最终数值，放 `BeforeDamageCalculation` 的 `final_damage`
- 如果需要拿到最终伤害值后再做额外行为，放 `BeforeDamageApplied`

### 不属于这四个伤害 Timing 的 legacy 实现

下面这些虽然出现在 `AttackLogic` 附近，但不应并入本文四个 timing：

- `至空至明`
  - 更像“被选为目标后”的冷却变化
- `飞向天际`
  - 更像位移 / 击退
- `六合劲`
  - 更像资源清零与节奏控制
- `幽居`
  - `素心神剑`
  - `左右互搏`
  - 更像追加攻击 / 追击 / 连击触发
- `笑傲江湖曲`
  - `清心普善咒`
  - `阿碧的歌声`
  - `吴侬软语`
  - 更像主动效果或团队技能，不属于单次伤害 timing

## 非目标

本文不解决以下问题：

- 毒伤、灼烧、自残是否复用同一套 timing
- 治疗是否需要独立 `BeforeHealApplied / OnHealed`
- 死亡前与死亡后是否需要单独 timing
- 多段伤害与一次技能多目标命中的事件聚合

这些问题后续再拆文档。

## 落地建议

- 当前 `OnDamageTaken` 中承载的命中拦截逻辑应迁出到 `BeforeHitResolved`
- 当前真实伤害 effect 后续应明确走“命中判定 -> 直接扣血 -> 扣血后触发”的窄管线
- `BeforeDamageApplied` 只保留真正依赖“本次伤害值”的防御与反制效果，不要再次混入闪避或普通攻防修正
